## Project Overview

Secure Policy Language (SPL) is a domain-specific programming language designed for defining granular, human-readable access control policies. The compiler performs lexical, syntactic, and semantic analysis, and integrates with LLM models to scan policies for security risks.

## Project Structure

- `lexer.py` - Lexical analyzer (tokenizer)
- `ast.py` - Abstract Syntax Tree node classes
- `grammar.ebnf` - Complete EBNF grammar specification
- `tokens.md` - Complete token list documentation
- `regex_patterns.md` - Regular expression patterns documentation
- `AST_DOCUMENTATION.md` - AST node classes documentation
